Fluxmon

Fluxmon is an attempt to build a monitoring system that does not require the configuration of thresholds.

Most monitoring systems these days operate using check plugins that measure a given set of values, compare them to configured thresholds, and return a status code accordingly. This approach is problematic, because everything relies on the configured thresholds to be sensible. Let's face it: Us admins are lazy, and more often than not, they aren't.

Fluxmon tries to get rid of configured thresholds by using statistical methods to analyze the measured data, see where regularities emerge, and use those for predictions.

Some features

  • OK/Critical thresholds emerge from measured data instead of being configured
  • Multiple hosts can submit measurements for a single target host
  • Measurements can be correlated
  • Configuration is kept on the measuring nodes and synced automatically
  • Checks are discovered and configured automatically

Some principles

  • Sensors deliver measurements only, they never evaluate anything
  • Deduced values are handled by the master
  • Meta data (e.g. existing parameters, their ranges and their units) are stored on the master, they are not reported by the sensor modules
  • Should credentials be necessary for a sensor to operate, they are stored in the measuring node's configuration only
